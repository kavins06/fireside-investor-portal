How to publish & manage deals

Everything you can do in Cowork, and how the workflow goes

You run the Fireside website by talking to Claude. Describe a deal — or hand over a pro forma — and Claude underwrites it, researches the market, and publishes it for you. You stay in control: you review and approve before anything goes live. This is the whole picture, start to finish.

03

Everything you can do

Add a deal

“Add a new deal to the website.”

Paste the details or attach a pro forma. Claude builds it, checks it, and publishes once you confirm.

See what’s live

“What deals are up right now?”

Claude lists every deal currently on the site.

Edit a live deal

“Update the Maple St deal — exit cap 6.5%.”

Claude pulls up the deal, makes the change, re-checks the numbers, and re-publishes it.

Soft-launch

“Publish it, but keep it off the homepage.”

It goes up as fundraising — reachable by direct link to share privately, but not listed on the home page.

Archive

“Close the Oak St deal, but keep its link.”

It drops off the home page; the page still works for anyone with the link. Nothing is deleted.

Remove

“Remove the test deal.”

Deleted from the site entirely — the page stops working. Good for test deals (still recoverable if ever needed).

04

Doing it right — so the deals are solid

Do

  • Give Claude the real source material — pro forma, OM, rent roll. The more it has, the less it has to assume.
  • Read the projected returns before you confirm. That number is your gut-check — if it looks wrong, it probably is.
  • Click a source link or two. Every market claim is backed by a real link — spot-check them.
  • Try anything new as “fundraising” first, look at the page, then make it active.
  • Keep your publish code private.

Don’t

  • Don’t publish a number you couldn’t defend to an investor. Claude researches hard, but you are the final set of eyes.
  • Don’t rush past the review. "Publish" should come after you’ve actually read it, not instead of reading it.
  • Don’t share the connector or your code with anyone you wouldn’t trust to publish to the live site.
  • Don’t paste your publish code into a public chat, email, or the website.
05

If something’s off

  • “missing or incorrect publish token” → your publish code is wrong. Ask whoever set you up for it again and paste it in.
  • Claude flags problems with the deal → they’re real (usually a missing number or source). Just answer its questions and it’ll fix and re-check.
  • A market finding won’t publish → it needs a real source link. Point Claude to where the figure came from.
  • You don’t see “Fireside Publish” → click the + / connectors button by the message box and switch it on.

First time? Two-minute setup

1. Add the connector once. In Claude: Settings → Connectors → Add custom connector, paste https://portal-eta-peach.vercel.app/api/mcp, leave auth blank, click Add. (A “not verified by Anthropic” note is normal for an in-house tool.)

2. Save your publish code. Tell Claude your publish code and say “remember this.” That code is what lets you publish — keep it private.

Cost: nothing extra to publish. The conversation runs on your own Claude plan; the connector and website cost nothing per deal.
